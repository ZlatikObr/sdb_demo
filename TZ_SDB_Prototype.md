# ТЗ: Кликабельный прототип SDB Dashboard (Авито Работа)

## 1. Цель и контекст

Демо-прототип Supply-Demand Balance Dashboard на mock-данных для встречи с командой Авито Работа (Этап 0 роадмэпа). Задача — продать концепцию: показать, как выглядит инструмент, и провести стейкхолдеров по 5 use cases, чтобы они предложили свои сценарии и уточнили скоуп до data request.

Не production-инструмент: без бэкенда, без реальных данных, без авторизации. Один HTML/React-файл, открывается локально или по ссылке, работает с ноутбука на проекторе.

## 2. Аудитория демо

Продукт, category management, pricing, коммерческая команда (CAM) Авито Работа. Прототип должен «говорить» на языке каждого: у каждого use case — свой экран или состояние.

## 3. Mock-данные

- 3–4 региона (Москва, МО, Казань, Екатеринбург) × дерево профессий 2–3 уровня (~15–20 листовых профессий: логистика — грузчик, сортировщик, курьер, водитель такси, водитель фуры; ритейл; общепит; строительство).
- Буферный пул как отдельный «микрорынок» с крупным объёмом.
- 12 месяцев истории для динамики м/м и г/г.
- Сегменты клиентов: SMB / Enterprise / агентства.
- Данные конкурента (HeadHunter) по вакансиям и клиентам.
- Данные должны быть **сюжетными**, а не случайными: заранее заложить перекосы под каждый use case (см. §5). Пример: избыток грузчиков + дефицит сортировщиков в МО; курьеры есть по всей МО, водителей фуры нет; перегретый сегмент SMB с высокой долей VAS в Казани.

## 4. Структура экранов

### 4.1 Главный экран — карта баланса
- Дерево профессий (сворачиваемое/разворачиваемое, 2–3 уровня) × селектор географии.
- Heatmap-раскраска узлов по балансу: перегрет (избыток supply) / сбалансирован / дефицит соискателей. Легенда обязательна.
- Ключевая метрика на узле: отклики на вакансию + дельта м/м.
- Фильтры: регион, сегмент клиента (SMB/Enterprise/агентства), период.
- Буферный пул визуально выделен отдельным блоком с объёмом и стрелками «куда можно перелить».

### 4.2 Карточка микрорынка (клик по узлу дерева)
Три блока метрик + монетизационный слой, все с дельтами м/м и г/г и сравнением с HH где применимо:
- **Demand:** активные вакансии, уникальные работодатели, рекламные бюджеты, зарплаты в вакансиях, разбивка по сегментам, разбивка по отраслям, вкладка «Топ-10 клиентов» (вакансии, выручка, динамика).
- **Supply:** уникальные соискатели, отклики, сплит интент/без интента, отклики на соискателя, средний срок поиска, зарплатные ожидания, топ-3 субститута, доля нашедших работу через платформу, CV-база (опционально, пометить как «Этап 2»).
- **Balance:** отклики на вакансию, соискатели на вакансию, доля VAS от выручки, сплит по монетизации (BBL / Listing Fee / CPR — в штуках и деньгах), средняя цена отклика, динамика баланса (линейный график), сравнение со средним по стране.

### 4.3 Экран сравнения микрорынков
Регион vs регион внутри профессии; профессия vs профессия внутри географии; каждый vs среднее по стране. Таблица + bar-чарты.

### 4.4 Экран Actions (рычаги)
Список рекомендаций, автоматически «сгенерированных» из данных, с привязкой к паттерну дисбаланса (реализация §7.1 пропоузала: каждому дисбалансу — управленческий action). Клик по рекомендации ведёт на соответствующий микрорынок.

## 5. Обязательные демо-сценарии (кликабельные пути)

Каждый сценарий — цепочка из 2–4 кликов, ведущая к «ага-моменту»:

1. **Переливка буфера (главный рычаг).** МО → логистика: грузчики перегреты, сортировщики в дефиците → карточка сортировщиков → рекомендация «перенастроить рекомендательную ленту, перелить N тыс. буферных соискателей». Показать стрелку перетока и прогнозный эффект.
2. **Совместная медийка.** Тот же перекос → рекомендация «предложить топ-клиенту X совместную кампанию про профессию сортировщика» с оценкой инкрементального revenue.
3. **CAM-routing.** Сравнение двух регионов с одинаковым балансом, но разной выручкой VAS → «Идите продавать VAS в регион Y» со списком клиентов.
4. **Pricing.** Сегмент с сигналами: мало откликов на вакансию + высокая доля VAS у SMB + преимущество над HH по откликам → рекомендация «кандидат на ценовой эксперимент». Обязательно показать распределение (не среднее) покупателей VAS и формулировку «сегмент для A/B-теста», не «поднять цену» (§7.4 пропоузала).
5. **Белые пятна.** Карта МО: курьеры есть везде, водители фуры — нет → «привлекать работодателей в микрорынок Z».

## 6. Интерактив: что кликабельно

Обязательно: раскрытие/схлопывание дерева, клик по узлу → карточка, переключение фильтров (регион, сегмент, период) с перерисовкой чисел, вкладки внутри карточки, переходы Action → микрорынок, тултипы на графиках. 

Достаточно fake-интерактива: фильтры могут переключать между заранее заготовленными состояниями. Не нужно: поиск, экспорт, настройки, реальные вычисления.

## 7. Технические требования

- Один самодостаточный файл (React/JSX или HTML+JS), mock-данные зашиты внутрь. Библиотеки чартов — recharts или Chart.js.
- Десктоп 1440px+, оптимизация под проектор: крупные шрифты, высокий контраст. Мобильная версия не нужна.
- Язык интерфейса — русский. Стиль — сдержанный аналитический (консалтинговый дашборд), нейтральная палитра + семантические цвета баланса. Без брендинга Авито (можно нейтральный акцентный цвет).
- Мгновенные переходы, без загрузок и лагов при кликах по дереву.

## 8. Вне скоупа

Бэкенд, реальные данные, авторизация, кластеризация по субститутам (Этап 4 — показать только как «coming soon»/топ-3 субститута в карточке), CV-слой (пометка «Этап 2»), сравнение с HH сверх мок-цифр.

## 9. Приёмка

- Все 5 демо-сценариев из §5 проходятся кликами без объяснений «представьте, что здесь…».
- Все метрики из §4.2 присутствуют и согласованы между экранами (числа не противоречат друг другу).
- Демо укладывается в 15 минут презентации.

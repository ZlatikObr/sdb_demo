# Infographic Deck

Референс-шаблон: плотный, но читаемый консалтинговый дек (тип питча Т-Виза). Задача скилла — воспроизводить его систему компонентов и раскладку, но в новой палитре под каждый проект.

## Порядок работы (строго)

1. Сначала контент, потом дизайн. Собери факты, цифры, источники (research / файлы / MCP). Инфографика оформляет проверенные данные, не выдумывает.
2. Составь storyline: 1 слайд = 1 сообщение (заголовок-вывод). Не пихай 4 анализа на один слайд.
3. Выбери палитру из банка (`assets/palettes.md`), зафиксируй ОДИН акцент на всю деку. Не бери жёлто-бирюзовую из образца.
4. Собери .pptx через `assets/pptx_toolkit.py` (редактируемо, шрифты встроены) или HTML-артборд 1280x720.
5. QA: отрендерь в картинки (soffice → pdf → pdftoppm), проверь переполнение, выравнивание, контраст. Один цикл фиксов.

## Дизайн-система

- Холст 1280x720 (16:9). Внешние поля 34-40px.
- Фон белый. Панели-блоки — светлый тёплый/холодный серый под палитру, скруглённые углы (радиус 12-14px).
- Верхняя строка вкладок-навигации на каждом контент-слайде: пилюли uppercase, активная залита акцентом, остальные светло-серые. Это wayfinding по разделам.
- Заголовок-вывод под навигацией: жирный дисплейный, левое выравнивание, формулирует «так что», а не тему. Над ним мелкий kicker (название раздела).
- Источник — мелким курсивом снизу под цифрами. Номер слайда — мелко в углу.
- Акцент используется точечно: активная вкладка, номера, highlight-боксы, ключевые числа, «выигрышная» ячейка. Не заливать акцентом всё.

## Типографика

- Дисплей (заголовки, числа): жирный гротеск. Дефолт Cabinet Grotesk (Extrabold/Bold). Встраивать в pptx.
- Текст: чистый гуманистический сан. Дефолт General Sans (Regular/Semibold/Italic). Встраивать.
- Заголовок слайда 26-30pt, разделы/подзаголовки 16-20pt, тело 13-15pt, подписи 9-11pt, крупные числа 44-72pt.
- Если шрифтов нет на машине — встраивай ttf в .pptx (см. toolkit `embed_fonts`), иначе fallback Arial.

## Библиотека компонентов (что и когда)

- Вкладки-навигация — на каждом контент-слайде, активна текущая секция.
- Крупный числовой callout — большое число + короткая подпись под ним. Для рынка, метрик, KPI.
- Highlight-бокс — рамка акцентом (толстая) на белом, внутри один ключевой вывод/стат. Для «главного числа» слайда.
- Нумерованный фиче-блок — круг с номером (акцент) + жирный подзаголовок + серое тело + highlight-бокс. Для «наше решение по шагам».
- Харви-матрица сравнения — строки-игроки/категории, колонки-критерии, шарики (полный/половина/пустой). Честно: не делать нашу строку сплошь полной, добавлять реальные слабые места. Заголовки колонок с подписью-расшифровкой.
- Donut-чарт — доля/структура (боли, сегменты). Нативный doughnut, фон plotArea в noFill.
- Bar-чарт — сравнение величин (размер рынка, RICE). Прямоугольники или нативный chart.
- 2x2 карта — риски или позиционирование (оси подписаны, точки/квадранты).
- Ценовые карточки — 2-3 тарифа: название, цена крупно, буллеты, рекомендация.
- Карточки-персоны — аватар (инициал в круге, не эмодзи в pptx), БОЛИ / МОТИВАЦИЯ, тег возраст/роль.
- RICE-таблица — Reach/Impact/Confidence/Effort + итог, цветные ячейки по величине.
- Process-steps — 1→2→3 блоки со стрелками (пайплайн/roadmap).

## Палитра (обязательно менять)

Образец жёлтый+бирюза. Всегда бери ДРУГУЮ из `assets/palettes.md` (кобальт, изумруд, оксблад, фиолет, тиловый графит, терракота). Один акцент на деку, светлый и тёмный тон под него, нейтральные серые. Не смешивать две акцентные системы.

## Анти-слоп правила

- Заголовок — вывод, а не список действий и не тема. Пассив и «проведён анализ» — плохо.
- Цифры только с источником. Нет выдуманных процентов опроса: если исследование качественное (N интервью) — счётчики «X из N», не «73%».
- В сравнении наш продукт не лучший во всём: показывай реальные слабые места, иначе читается как реклама.
- Одно сообщение на слайд. Плотный многопанельный слайд допустим только для обзорного (ЦА/рынок), и то аккуратно.
- Без ё и длинного тире в тексте слайдов (дефис или переформулировка).
- Эмодзи в pptx не рендерятся цветом — вместо аватаров-эмодзи ставь инициал в круге.

## Вывод и QA

- Редактируемый pptx: `assets/pptx_toolkit.py` даёт helpers (text/rect/oval/ball/donut/navtabs/big-number/highlight/embed_fonts). Шарики — фигуры, шрифты встроены.
- HTML: артборд 1280x720 со scale-fit и @media print (экспорт в PDF из Chrome).
- QA обязательный: `soffice --headless --convert-to pdf`, затем `pdftoppm -jpeg -r 150`, просмотреть, поправить переполнения. Для проверки шрифта поставить ttf в ~/.fonts и fc-cache.

Подробные рабочие функции — в `assets/pptx_toolkit.py`. Банк палитр — в `assets/palettes.md`.

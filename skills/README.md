# skills/

Экспорт всех Claude-скиллов, установленных у пользователя на момент выгрузки (2026-07-06). Скиллы — это модули с инструкциями и вспомогательными файлами (скрипты, шаблоны, референсы), которые Claude подключает при выполнении задач определённого типа.

## Структура папки

```
skills/
├── <skill-name>/
│   ├── SKILL.md         # обязательный файл: имя, description (триггеры), сама инструкция
│   ├── scripts/         # вспомогательные скрипты (если есть)
│   ├── references/      # справочные материалы (если есть)
│   └── assets/          # шаблоны, статические файлы (если есть)
├── marketing/           # плагин с маркетинговыми скиллами, каждый — своя подпапка
│   └── <skill-name>/
│       └── SKILL.md
└── README.md            # этот файл
```

Каждый скилл — самостоятельная папка с `SKILL.md` в корне. Claude читает `description` из фронтматтера файла, чтобы понять, когда скилл нужно подключать, а тело файла — как именно действовать. Папки `scripts/`, `references/`, `assets/`, `workflows/`, `agents/` внутри скилла — вспомогательные материалы, на которые ссылается сам `SKILL.md`.

Скиллы делятся на две группы:
- **корневые** — самостоятельные скиллы верхнего уровня;
- **marketing/** — набор скиллов одного плагина (marketing), сгруппированных в подпапку, так как они устанавливаются и обновляются вместе.

## Каталог скиллов

### Дизайн и фронтенд

- **design-motion-principles** — эксперт по motion-дизайну (методики Emil Kowalski, Jakub Krehel, Jhey Tompkins). Два режима: создание интерактивных компонентов с осмысленной анимацией или аудит существующих анимаций на «AI-slop» паттерны с выдачей HTML-отчёта.
- **design-taste-frontend** — анти-шаблонный скилл для лендингов, портфолио и редизайнов. Сам определяет дизайн-направление по брифу и не допускает «типового» вида интерфейса.
- **high-end-visual-design** — задаёт шрифты, отступы, тени, структуру карточек и анимации уровня дорогих агентств; блокирует дефолтные решения, которые делают AI-дизайн дешёвым на вид.
- **impeccable** — универсальный скилл для дизайна, редизайна, критики и полировки любого фронтенд-интерфейса: UX-ревью, иерархия, доступность, типографика, motion, копирайтинг интерфейса, edge-кейсы, дизайн-системы.
- **industrial-brutalist-ui** — стиль «индустриальный брутализм»: swiss-типографика + военные терминалы, жёсткие сетки, аналоговая деградация. Для дашбордов и порталов с «рассекреченным» видом.
- **minimalist-ui** — чистый редакторский стиль: тёплая монохромная палитра, типографический контраст, плоские bento-сетки, приглушённые пастельные тона, без градиентов и теней.
- **redesign-existing-projects** — апгрейд существующих сайтов/приложений: аудит текущего дизайна, поиск типовых AI-паттернов, применение премиальных стандартов без поломки функциональности.

### Документы и презентации

- **docx** — создание, чтение и редактирование Word-документов (.docx): оглавления, заголовки, колонтитулы, замена изображений, работа с правками и комментариями.
- **pptx** — работа с презентациями (.pptx): создание, чтение, редактирование слайдов, шаблонов, спикерских заметок.
- **pdf** — работа с PDF: извлечение текста и таблиц, слияние/разбиение файлов, повороты страниц, водяные знаки, формы, шифрование, OCR.
- **xlsx** — работа с Excel/таблицами (.xlsx, .csv, .tsv): формулы, форматирование, анализ данных, диаграммы, конвертация форматов.
- **infographic-deck** — инфографические слайды и презентации в стиле консалтингового питч-дека: навигация вкладками, callout-цифры, харви-матрицы, донат/бар-чарты, RICE-таблицы, 2x2-карты. Отдаёт редактируемый .pptx или HTML.

### Текст и стратегия

- **russian-final-style** — приведение русского текста к стилю McKinsey/BCG для аналитических отчётов: без длинных тире, без «не X, а Y», без канцелярита и нейросетевых усилителей, с тезисом в начале абзаца.
- **strategy-consultant** — стратегический консультант уровня McKinsey/BCG для топ-менеджмента, отвечает на русском. Запускается только по явной команде «консультируй».

### Служебные и мета-скиллы

- **skill-creator** — создание новых скиллов, редактирование и оптимизация существующих, прогон эвалов и бенчмарков.
- **consolidate-memory** — рефлексивный проход по файлам памяти: объединение дублей, исправление устаревших фактов, чистка индекса.
- **schedule** — создание и изменение автоматических задач по расписанию («каждое утро», «напомни через час» и т.п.).
- **setup-cowork** — гид по первоначальной настройке Cowork: подбор плагинов под роль, подключение инструментов, пробный запуск скилла.

### marketing/ (плагин, 8 скиллов)

- **brand-review** — проверка контента на соответствие брендвойсу и стайлгайду, с флагами по степени серьёзности и конкретными правками.
- **campaign-plan** — полный бриф кампании: цели, аудитория, месседжинг, каналы, контент-календарь, метрики успеха.
- **competitive-brief** — исследование конкурентов, сравнение позиционирования, поиск разрывов и угроз для battlecards.
- **content-creation** — черновики маркетингового контента по каналам (базовый фреймворк, используется как основа для draft-content).
- **draft-content** — черновики постов, писем, лендингов, пресс-релизов с форматированием под конкретный канал и SEO-рекомендациями.
- **email-sequence** — многошаговые email-цепочки: копирайт, тайминг, ветвление логики, условия выхода, A/B-тесты.
- **performance-report** — отчёт по эффективности маркетинга: метрики, тренды, успехи/провалы, приоритеты на следующий период.
- **seo-audit** — комплексный SEO-аудит: ключевые слова, on-page анализ, контентные разрывы, технические проверки, сравнение с конкурентами.

## Как это работает у Claude

`SKILL.md` каждого скилла содержит фронтматтер с полями `name` и `description`. Claude сопоставляет запрос пользователя с `description` и подключает подходящий скилл, после чего следует инструкциям в теле файла и при необходимости использует приложенные скрипты/шаблоны. Скиллы не исполняются автоматически при каждом запросе — только когда запрос пользователя соответствует триггерам, описанным в `description`.
